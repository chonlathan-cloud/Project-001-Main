# User Manual for Subcontractor

## 1. วัตถุประสงค์ของระบบ

ระบบนี้ช่วยให้ผู้รับเหมา / ผู้รับเหมาช่วงสามารถส่งคำขอเบิกหรือบันทึกรายการที่เกี่ยวข้องกับโครงการได้สะดวกขึ้น โดยมีการอัปโหลด receipt และส่งข้อมูลให้ Admin ตรวจสอบและอนุมัติ

## 2. สิทธิ์ของผู้ใช้งานประเภท Subcontractor

ผู้ใช้งานประเภท Subcontractor ใช้งานเมนูหลักได้ 2 ส่วน:

- `Input`
- `Profile`

หมายเหตุ:

- โครงการที่ท่านเห็นในหน้า Input จะขึ้นอยู่กับการกำหนดสิทธิ์จาก Admin

## 3. การเข้าสู่ระบบ

### 3.1 วิธีเข้าสู่ระบบ

1. เปิดหน้า `Login`
2. เลือก `Continue as Subcontractor with LINE`
3. เข้าสู่ระบบด้วยบัญชี LINE

### 3.2 กรณีเข้าใช้งานครั้งแรก

หากเป็นการเข้าใช้งานครั้งแรก ระบบจะพาท่านไปยังหน้า `Subcontractor Sign Up`

## 4. การลงทะเบียนครั้งแรก

หน้า `Subcontractor Sign Up` ใช้สำหรับลงทะเบียนและยืนยันตัวตนเบื้องต้น

### 4.1 ข้อมูลที่ต้องกรอก

- Company / Subcontractor Name
- Default Contact Name
- Default Phone Number
- Tax Identification Number
- Default Bank Name
- Default Account Number
- Default Account Name
- KYC ID Card Image

### 4.2 วิธีใช้งาน

1. ตรวจสอบว่า LINE UID ถูกเชื่อมกับการสมัครแล้ว
2. กรอกข้อมูลให้ครบ
3. แนบรูปบัตรประชาชนหรือเอกสาร KYC ตามที่ระบบกำหนด
4. กดส่งแบบฟอร์ม

### 4.3 หลังลงทะเบียนสำเร็จ

ระบบจะบันทึกข้อมูลโปรไฟล์เริ่มต้นของท่าน และพาเข้าสู่หน้าที่ใช้งานได้ทันที

## 5. หน้า Input

หน้า `Input` เป็นหน้าหลักสำหรับส่งคำขอเข้าไปยังระบบ

### 5.1 จุดประสงค์ของหน้า Input

ใช้สำหรับส่งข้อมูลรายการเบิกหรือรายการรับเงิน พร้อมแนบ receipt เพื่อให้ Admin ตรวจสอบ

### 5.2 ขั้นตอนการใช้งาน

1. อัปโหลดรูปหรือ PDF ของบิล / ใบเสร็จ
2. ระบบจะอ่านข้อมูลจากเอกสารให้อัตโนมัติในบางช่อง
3. เลือกประเภท `รายจ่าย` หรือ `รายรับ`
4. เลือก `โครงการ`
5. ตรวจสอบข้อมูลที่ระบบกรอกให้
6. กรอกหรือแก้ไขข้อมูลที่ยังขาด
7. กด `ส่ง`

### 5.3 ข้อมูลที่ต้องกรอกในแบบฟอร์ม

- โครงการ
- ชื่อ - นามสกุล
- เบอร์ติดต่อ
- วัน/เดือน/ปี
- เลขที่ใบเสร็จ
- วันที่เอกสาร
- ประเภทงาน
- ประเภทการเบิก
- ผู้ขาย / ร้านค้า
- รายละเอียดเพิ่มเติม
- ธนาคาร
- จำนวนเงินที่ขอเบิก
- เลขที่บัญชี
- ชื่อบัญชี

### 5.4 ข้อมูลที่ระบบช่วยกรอกให้อัตโนมัติ

ระบบจะใช้ข้อมูลจากโปรไฟล์ของท่านเพื่อช่วยกรอกค่าเริ่มต้นในบางช่อง เช่น:

- ชื่อผู้ติดต่อ
- เบอร์โทร
- ธนาคาร
- เลขบัญชี
- ชื่อบัญชี

### 5.5 การเลือกโครงการ

- ท่านจะเห็นเฉพาะโครงการที่ Admin กำหนดให้เท่านั้น
- หากไม่พบโครงการใดเลย แปลว่ายังไม่มี Assigned Project สำหรับบัญชีนี้

### 5.6 หลังส่งสำเร็จ

ระบบจะแสดง:

- Request ID
- สถานะของรายการ
- สรุปข้อมูลที่ส่ง
- ตัวอย่างไฟล์ receipt

### 5.7 ข้อควรระวัง

- หากระบบอ่านข้อมูลจาก receipt ไม่ครบ ควรตรวจทานและแก้ไขก่อนส่ง
- ควรตรวจสอบจำนวนเงินและเลขที่ใบเสร็จทุกครั้ง

## 6. สถานะของคำขอ

หลังส่งคำขอแล้ว คำขอจะเข้าสู่กระบวนการตรวจสอบของ Admin

สถานะที่อาจพบได้ เช่น:

- `PENDING_ADMIN` รอผู้ดูแลตรวจสอบ
- `APPROVED` อนุมัติแล้ว
- `PAID` จ่ายเงินแล้ว
- `REJECTED` ไม่อนุมัติ

## 7. Profile

หน้า `Profile` ใช้สำหรับดูข้อมูลของบัญชีผู้ใช้งาน

### 7.1 สิ่งที่แสดงในหน้า Profile

- รูปโปรไฟล์
- ชื่อผู้ใช้งาน
- บทบาทของผู้ใช้งาน
- สถิติการใช้งานบางส่วน

### 7.2 การเปลี่ยนรูปโปรไฟล์

ถ้าระบบเปิดใช้งานรูปจาก LINE หรือรูปที่อัปโหลดเอง ท่านสามารถ:

- กด `Change Photo` เพื่ออัปโหลดรูปใหม่
- กด `Use LINE Avatar` เพื่อกลับไปใช้รูปจาก LINE

หมายเหตุ:

- รูปโปรไฟล์ไม่ใช่เอกสาร KYC
- เอกสาร KYC จะถูกเก็บแยกต่างหาก

## 8. การออกจากระบบ

1. กด `Sign Out` จากเมนูด้านซ้าย
2. ระบบจะออกจาก session และกลับไปหน้า Login

## 9. ปัญหาที่พบบ่อย

### 9.1 เข้าสู่ระบบด้วย LINE แล้วเข้าไม่ได้

- ตรวจสอบว่า LINE login สำเร็จ
- ลองปิดหน้าแล้วเข้าสู่ระบบใหม่
- หากเป็นครั้งแรก ให้ตรวจสอบว่าท่านกรอกแบบฟอร์มสมัครครบหรือยัง

### 9.2 ไม่เห็นโครงการในหน้า Input

- เป็นไปได้ว่า Admin ยังไม่ได้กำหนดสิทธิ์โครงการให้
- ให้ติดต่อ Admin เพื่อกำหนด Assigned Projects

### 9.3 อัปโหลดไฟล์ receipt ไม่ได้

- ตรวจสอบว่าไฟล์เป็นรูปภาพหรือ PDF
- ตรวจสอบขนาดไฟล์และความชัดเจนของเอกสาร

### 9.4 ข้อมูลที่ระบบอ่านจากเอกสารไม่ถูกต้อง

- ตรวจสอบและแก้ไขด้วยตนเองก่อนกดส่ง
- เอกสารที่ชัดเจนจะช่วยให้ระบบอ่านข้อมูลได้ดีขึ้น

## 10. คำแนะนำในการใช้งาน

- ควรใช้รูปหรือ PDF ที่ชัดเจน
- ควรกรอกข้อมูลบัญชีให้ถูกต้องตั้งแต่ครั้งแรก เพื่อให้ระบบช่วยกรอกอัตโนมัติในครั้งถัดไป
- ควรตรวจสอบข้อมูลทุกครั้งก่อนกดส่ง

## 11. เมื่อต้องการความช่วยเหลือ

หากพบปัญหา ให้แจ้ง Admin พร้อมข้อมูลดังนี้:

- ชื่อโครงการ
- วันเวลาที่พบปัญหา
- หมายเลขคำขอ (ถ้ามี)
- ภาพหน้าจอของปัญหา
