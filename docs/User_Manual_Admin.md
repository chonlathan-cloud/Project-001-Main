# User Manual for Admin

## 1. วัตถุประสงค์ของระบบ

ระบบนี้ใช้สำหรับบริหารโครงการก่อสร้าง โดยฝั่งผู้ใช้งานภายในจะแบ่งเป็น 2 บทบาท:

- `Owner`: สิทธิ์สูงสุด ใช้งานได้ทุกหน้าและทำรายการอนุมัติ/แก้ไขข้อมูลสำคัญได้
- `Admin`: สิทธิ์สำหรับดูและตรวจสอบข้อมูลปฏิบัติการ ไม่สามารถดู `Dashboard` และ `Chat AI` และไม่สามารถอนุมัติหรือเปลี่ยนสถานะคำขอได้

โดยรวมระบบรองรับการทำงานต่อไปนี้:

- เข้าดูภาพรวมของโครงการและสถานะการเงิน
- สร้างและจัดการโครงการ
- เชื่อมข้อมูล BOQ
- ตรวจสอบคำขอเบิกจากผู้รับเหมา
- ค้นหาข้อมูลเชิงลึกจากคลังข้อมูล
- ดูข้อมูลโปรไฟล์และตั้งค่าผู้ใช้งาน
- ใช้งาน Chat AI เพื่อช่วยวิเคราะห์ข้อมูล

## 2. การเข้าสู่ระบบ

### 2.1 วิธีเข้าสู่ระบบ

1. เปิดหน้า `Login`
2. เลือก `Continue as Admin with Google`
3. ลงชื่อเข้าใช้ด้วยบัญชี Google ที่ได้รับสิทธิ์เป็นผู้ดูแลระบบ
4. เมื่อเข้าสู่ระบบสำเร็จ ระบบจะพาไปยังหน้าเริ่มต้นตามบทบาทของผู้ใช้

### 2.2 หากเข้าสู่ระบบไม่ได้

- ตรวจสอบว่าอีเมล Google ของท่านได้รับสิทธิ์เป็น `Owner` หรือ `Admin` แล้ว
- ตรวจสอบว่าเปิดใช้งาน popup ของ browser แล้ว
- หากระบบแจ้งสิทธิ์ไม่เพียงพอ ให้ติดต่อผู้ดูแลระบบหลัก

## 3. บทบาทและเมนูหลัก

### 3.1 Owner

`Owner` จะเห็นเมนูเต็ม:

- `Dashboard`
- `Project`
- `Insights`
- `Input`
- `Approval`
- `Chat AI`
- `Profile`
- `Setting`

### 3.2 Admin

`Admin` จะเห็นเฉพาะเมนูที่ดูหรือตรวจสอบได้:

- `Project`
- `Insights`
- `Input`
- `Approval`
- `Profile`
- `Setting`

ข้อจำกัดของ `Admin`:

- ไม่เห็นและเข้าใช้งานหน้า `Dashboard` ไม่ได้
- ไม่เห็นและเข้าใช้งานหน้า `Chat AI` ไม่ได้
- ดูรายการใน `Approval` ได้ แต่ไม่สามารถ `Approve`, `Reject`, `Mark Paid` หรือแก้ไขข้อมูลคำขอได้
- ดูข้อมูลโครงการและ BOQ ได้ แต่ไม่สามารถสร้าง/แก้ไขโครงการหรือ Sync BOQ ได้
- ดูข้อมูล Settings ได้ แต่ไม่สามารถแก้ไขผู้รับเหมา รีเซ็ต LINE binding หรือจัดการสิทธิ์ผู้ดูแลระบบได้

## 4. Dashboard

หน้า `Dashboard` ใช้สำหรับดูภาพรวมการดำเนินงานและการเงินของระบบ

สิทธิ์: ใช้ได้เฉพาะ `Owner`

### 4.1 สิ่งที่แสดงในหน้า Dashboard

- งบรวมของโครงการทั้งหมด
- ต้นทุนที่บันทึกแล้ว
- จำนวนรายการที่รออนุมัติ
- Profit Margin
- Monthly Cashflow
- Zone Status
- Budget Health
- Top Risky Projects
- Attention Items
- Recent Actions

### 4.2 วิธีใช้งาน

- ใช้ดูภาพรวมว่าเงินอยู่ตรงไหน และงานติดอยู่ตรงไหน
- ใช้ `Top Risky Projects` เพื่อดูว่าโครงการใดมีความเสี่ยงสูงจากยอดค้างชำระหรือรายการรออนุมัติ
- ใช้ `Recent Actions` เพื่อตรวจสอบกิจกรรมล่าสุดของทีม

## 5. Project

หน้า `Project` ใช้สำหรับสร้างและจัดการโครงการ

สิทธิ์:

- `Owner`: ดู สร้าง แก้ไข และ Sync BOQ ได้
- `Admin`: ดูรายการโครงการ รายละเอียดโครงการ และ BOQ ได้เท่านั้น

### 5.1 สิ่งที่ทำได้

- เพิ่มโครงการใหม่
- เปลี่ยนชื่อโครงการ
- ดูงบรวมของแต่ละโครงการ
- เปิดรายละเอียดโครงการ
- เชื่อม BOQ ของโครงการ

### 5.2 การเพิ่มโครงการ

1. กด `Add new project`
2. กรอกข้อมูลโครงการ
3. บันทึกข้อมูล

### 5.3 การเชื่อม BOQ

ระบบรองรับ BOQ 2 ฝั่งต่อ 1 โครงการ:

- `Customer BOQ`
- `Subcontractor BOQ`

การเชื่อม BOQ ทำ 2 รอบ:

1. เชื่อม `Customer BOQ`
2. เชื่อม `Subcontractor BOQ`

### 5.4 การเปิดรายละเอียดโครงการ

คลิกการ์ดของโครงการเพื่อเข้าสู่หน้า `Project Detail`

## 6. Project Detail

หน้า `Project Detail` ใช้สำหรับเปรียบเทียบ BOQ ของโครงการในเชิงลึก

### 6.1 มุมมองที่มีให้ใช้

- `Compare`
- `Customer`
- `Subcontractor`

### 6.2 สิ่งที่แสดงในหน้า Project Detail

- สรุปงบของ Customer และ Subcontractor
- Variance และ Margin
- กราฟสรุปตามหมวดงาน
- BOQ Tree แบบลำดับชั้น
- ข้อมูล execution summary ของโครงการ

### 6.3 ประโยชน์ของหน้า Project Detail

- ใช้เปรียบเทียบต้นทุนระหว่างสองฝั่ง
- ใช้หาหมวดงานที่มี margin สูงหรือต่ำ
- ใช้เปิดดูรายละเอียดระดับ WBS

## 7. Insights

หน้า `Insights` เป็นคลังข้อมูลรวมสำหรับค้นหาและวิเคราะห์ข้อมูลจากหลายแหล่งในระบบ

### 7.1 ข้อมูลที่รวมอยู่ในหน้า Insights

- Input Requests
- Installments
- Transactions

### 7.2 สิ่งที่ทำได้

- ค้นหาข้อมูลแบบรวม
- กรองตามโครงการ
- กรองตามสถานะ
- กรองตามช่วงเวลา
- ดูเฉพาะรายการซ้ำ
- ดูเฉพาะรายการเกินกำหนด
- Export ข้อมูลออกเป็นไฟล์

### 7.3 เหมาะสำหรับกรณีใด

- ใช้ตรวจสอบรายการผิดปกติ
- ใช้ค้นหาประวัติรายการย้อนหลัง
- ใช้เปิดต่อไปยังหน้า Approval หรือ Project Detail

## 8. Input

หน้า `Input` ใช้สำหรับสร้างคำขอใหม่เข้าสู่ระบบ แม้ Admin ก็สามารถใช้งานหน้านี้ได้

### 8.1 ขั้นตอนการใช้งาน

1. อัปโหลดรูปหรือ PDF ของบิล/ใบเสร็จ
2. ระบบจะพยายามอ่านข้อมูลจากเอกสาร
3. เลือกประเภท `รายรับ` หรือ `รายจ่าย`
4. เลือกโครงการ
5. ตรวจสอบและกรอกข้อมูลที่จำเป็น
6. กด `ส่ง`

### 8.2 ข้อมูลสำคัญที่ต้องตรวจสอบก่อนส่ง

- โครงการ
- ชื่อผู้ขอ
- เบอร์ติดต่อ
- วันที่เอกสาร
- เลขที่ใบเสร็จ
- ผู้ขาย / ร้านค้า
- จำนวนเงิน
- ข้อมูลธนาคาร

### 8.3 หลังส่งสำเร็จ

ระบบจะแสดง:

- Request ID
- สถานะของคำขอ
- สรุปข้อมูลที่ส่ง
- ตัวอย่างไฟล์ receipt

## 9. Approval

หน้า `Approval` เป็นหน้าหลักสำหรับตรวจสอบและอนุมัติคำขอจากหน้า Input

สิทธิ์:

- `Owner`: ดู แก้ไข อนุมัติ ปฏิเสธ และ Mark Paid ได้
- `Admin`: ดูรายการ รายละเอียด และ receipt preview ได้เท่านั้น

### 9.1 ส่วนประกอบของหน้า Approval

- Review Queue
- ฟิลเตอร์ตามสถานะ ประเภท และโครงการ
- รายละเอียดคำขอ
- Receipt Preview
- OCR warnings
- ปุ่มดำเนินการ

### 9.2 วิธีการอนุมัติคำขอ

ใช้ได้เฉพาะ `Owner`

1. เลือกรายการจาก `Review Queue`
2. ตรวจสอบข้อมูลของคำขอ
3. เปิดดู receipt หากจำเป็น
4. ปรับแก้ข้อมูลได้หากสถานะยังแก้ไขได้
5. กด `Approve`

### 9.3 วิธีปฏิเสธคำขอ

ใช้ได้เฉพาะ `Owner`

1. เลือกรายการ
2. ใส่ `Review Note`
3. กด `Reject`

### 9.4 วิธีบันทึกว่าได้จ่ายแล้ว

ใช้ได้เฉพาะ `Owner`

1. เปิดคำขอที่มีสถานะ `Approved`
2. กรอก `Payment Reference` หากมี
3. กด `Mark Paid`

### 9.5 ข้อควรสังเกต

- หากระบบขึ้น `Duplicate Red Flag` ควรตรวจสอบว่ามีรายการซ้ำหรือไม่
- หากระบบขึ้น `OCR Low Confidence` ควรตรวจข้อมูลจาก receipt ด้วยตนเอง
- `Technical details` ใช้สำหรับดูข้อมูล OCR debug เมื่อจำเป็น ไม่ใช่ส่วนที่ต้องใช้ทุกครั้ง

## 10. Chat AI

หน้า `Chat AI` ใช้สำหรับถามคำถามเกี่ยวกับข้อมูลในระบบ

สิทธิ์: ใช้ได้เฉพาะ `Owner`

### 10.1 ตัวอย่างการใช้งาน

- ถามว่ามีรายการใดค้างชำระ
- ถามว่าโครงการใดมีความเสี่ยงสูง
- ถามหาข้อมูลที่เกี่ยวข้องกับ Project Detail

### 10.2 คำแนะนำ

- ถามเป็นประโยคสั้นและชัดเจน
- หากต้องการวิเคราะห์เฉพาะโครงการ ควรระบุชื่อโครงการในคำถาม

## 11. Profile

หน้า `Profile` ใช้สำหรับดูข้อมูลผู้ใช้งานปัจจุบัน

### 11.1 สิ่งที่แสดง

- รูปโปรไฟล์
- ชื่อผู้ใช้
- บทบาทของผู้ใช้
- สถิติการใช้งานบางส่วน

## 12. Setting

หน้า `Setting` ใช้สำหรับจัดการข้อมูลสำคัญในระบบ

สิทธิ์:

- `Owner`: ดูและแก้ไขข้อมูล Settings ได้
- `Admin`: ดูข้อมูล Settings ได้เท่านั้น

### 12.1 สิ่งที่ทำได้

- จัดการข้อมูลผู้รับเหมา
- ดู KYC ของผู้รับเหมา
- รีเซ็ต LINE binding
- กำหนด `Assigned Projects`
- แก้ไขข้อมูลบัญชีธนาคาร
- จัดการรายชื่อ Admin
- ดูสถานะ Integrations แบบ read-only

### 12.2 Assigned Projects คืออะไร

โครงการที่เลือกไว้ในส่วนนี้ จะเป็นโครงการที่ Subcontractor คนนั้นเห็นได้ในหน้า Input เท่านั้น

### 12.3 การจัดการ Admin

- เพิ่มผู้ใช้งานภายในด้วยอีเมล
- เปลี่ยนชื่อที่แสดง
- กำหนดบทบาทเป็น `Owner` หรือ `Admin`
- เปิด/ปิดสถานะการใช้งาน

ข้อควรระวัง:

- ผู้ใช้งานภายในที่เข้าใช้งานครั้งแรกจากอีเมลที่ระบบอนุญาต จะถูกสร้างเป็น `Admin` โดยค่าเริ่มต้น
- สิทธิ์ `Owner` ต้องมาจากข้อมูลใน Firestore ที่ตั้ง `role` เป็น `owner` หรือจาก `Owner` คนอื่นที่เปลี่ยนบทบาทให้
- ต้องมี `Owner` ที่เปิดใช้งานอยู่ในระบบอย่างน้อย 1 คนเสมอ
- ผู้ใช้งานไม่สามารถเปลี่ยนบทบาทหรือสถานะการใช้งานของตัวเองได้
- `Admin` ไม่สามารถเพิ่ม/แก้ไขผู้ใช้งานภายในหรือเปลี่ยนบทบาทของใครได้

### 12.4 Integrations

- ใช้สำหรับดูสถานะการเชื่อมต่อ Backend/API, Google Cloud, Firebase, LINE/LIFF, GCS, AI services และ Admin Access
- หน้านี้เป็น read-only ในรอบแรกของการพัฒนา frontend
- ระบบจะแสดงเฉพาะสถานะและค่าที่ไม่ใช่ความลับ เช่น model name, bucket name หรือจำนวนอีเมลที่อนุญาต
- ระบบจะไม่แสดง secret, API key, credential path หรือ token จริงบนหน้า frontend

## 13. การออกจากระบบ

1. กด `Sign Out` ที่ด้านล่างของเมนูด้านซ้าย
2. ระบบจะออกจาก session ปัจจุบันและกลับไปหน้า Login

## 14. คำแนะนำการใช้งาน

- `Owner` ควรตรวจสอบข้อมูลจาก receipt ก่อนอนุมัติทุกครั้ง
- `Owner` ควรใช้ Dashboard เพื่อดูภาพรวม และใช้ Insights สำหรับค้นหาข้อมูลเชิงลึก
- ควรกำหนด Assigned Projects ให้ผู้รับเหมาก่อนใช้งานหน้า Input
- ควรตรวจสอบสิทธิ์ของ `Owner` และ `Admin` เป็นระยะ

## 15. ปัญหาที่พบบ่อย

### 15.1 เข้าระบบไม่ได้

- ตรวจสอบสิทธิ์ของบัญชี Google
- ตรวจสอบว่า browser ไม่บล็อก popup

### 15.2 ไม่เห็นข้อมูลในหน้า Approval

- ตรวจสอบฟิลเตอร์ที่เลือกไว้
- ตรวจสอบว่าโครงการหรือสถานะที่ต้องการมีข้อมูลจริง

### 15.3 เข้า Dashboard หรือ Chat AI ไม่ได้

- ตรวจสอบว่าบัญชีของท่านเป็น `Owner`
- หากเป็น `Admin` จะไม่สามารถใช้งาน 2 หน้านี้ได้ตามนโยบายสิทธิ์

### 15.4 กด Approve / Reject / Mark Paid ไม่ได้

- ตรวจสอบว่าบัญชีของท่านเป็น `Owner`
- หากเป็น `Admin` จะดูรายการได้เท่านั้น แต่ไม่สามารถเปลี่ยนสถานะคำขอได้

### 15.5 Subcontractor ไม่เห็นโครงการในหน้า Input

- ตรวจสอบ Assigned Projects ในหน้า Setting

### 15.6 Receipt เปิดไม่ได้

- ตรวจสอบว่าไฟล์ถูกอัปโหลดสำเร็จหรือไม่
- ลองใช้ปุ่ม `Open Receipt`

## 16. ช่องทางภายในสำหรับขอความช่วยเหลือ

หากพบปัญหาในการใช้งาน ให้ส่งข้อมูลต่อไปนี้แก่ทีมดูแลระบบ:

- ชื่อหน้า
- วันที่และเวลาที่พบปัญหา
- Request ID หรือ Project Name ที่เกี่ยวข้อง
- ภาพหน้าจอของปัญหา
